# 常見問題

## 一般問題

### 什麼是 DEE？

DEE（Database Engineering Essentials，資料庫工程精要）是一系列有編號的資料庫設計指引。完整概述請參閱 [DEE-0](0)。

### DEE 的目標讀者是誰？

與資料庫打交道的應用程式開發者——從學習 schema 設計的初學者，到需要查閱分片和遷移策略等進階主題的資深工程師。

### DEE 是否針對特定資料庫？

不是。DEE 原則在可能的範圍內與資料庫無關。當某項原則特定於某個資料庫引擎（例如 PostgreSQL GIN 索引）時，會明確標註。

### 我可以提出新的 DEE 嗎？

可以。請按照 [DEE-0](0) 中描述的結構提交 pull request。

## 內容問題

### 為什麼使用 RFC 2119 關鍵字（MUST、SHOULD、MAY）？

這些關鍵字提供精確的指引層級。MUST 表示該原則不可妥協。SHOULD 表示除非有書面理由，否則應遵循。MAY 表示可選。

### DEE 編號是如何分配的？

DEE 按類別範圍編號。完整對應請參閱 [DEE-0](0) 中的分類表。

### 如果我的設計需要偏離某個 DEE 怎麼辦？

記錄原因。DEE 是指引，不是法律。目標是做出有根據的決策，而非盲目遵守規則。
